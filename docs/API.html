    获取指定id的任务

    获得指定任务的id，post方式，URL为https://littlefish33.cn:8080/questionnaire/select，表单字段id
    成功结果为任务的json字符串

    POST

    请求 Content-Type

    创建新的任务

    创建新的任务，POST请求，需要携带token和data字段，data字段表示任务的json字符串，创建成功之后会返回一个json字符串，里面包含着具体的创建的任务的ID（可以参考下面的成功示例）；
    需要注意的是：
    1. 表单字段data的json字符串中的creator必须与token所代表的用户一致；
    2. 表单字段data的json字符串中的taskID实际上是一个无用的选项，因为taskID是后台唯一生成的，所以在提交的时候提交taskID为一个空字符串就好，可以参考下面的成功示例（其实就算是乱七八糟的字符串也无所谓，因为后台不会用的）

    接口URL

    https://littlefish33.cn:8080/questionnaire/create

    请求方式

    POST

    请求 Content-Type

    multipart/form-data

    请求原始Header

    参数 示例值 参数描述
    Accept */* 无描述

    更新用户信息

    更新用户信息接口，https://littlefish33.cn:8080/user/update，需提供token和表单字段
    注意：
    1. 除了phone必须在数据库中唯一和phone必须和token对应的phone保持一致外，表单字段的检查后端并没有提供，比如说age必须为int，奶牛不用填university等字段，phone不能为空等，因此这些需要在前端代码中先检测在提交
